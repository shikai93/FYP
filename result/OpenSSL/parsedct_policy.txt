Number of seconds in the future that an SCT timestamp can be, by default, without being considered invalid. This is added to time() when setting a default value for CT_POLICY_EVAL_CTX.epoch_time_in_ms. It can be overridden by calling CT_POLICY_EVAL_CTX_set_time().

time(NULL) shouldn't ever fail, so don't bother checking for -1.
